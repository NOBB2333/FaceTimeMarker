# FaceTimeMarker 产品设计文档

最后更新：2026-07-07

## 产品定位

FaceTimeMarker 是一个本地离线的视频人物档案工作台。它不是剪辑软件本体，也不是训练平台，而是介于“视频分析工具”和“素材管理系统”之间：

```text
视频素材 -> 自动识别人脸与人物时间轴 -> 人工复核整理 -> 跨作品人物档案 -> 参考图/四视图资产
```

它要解决的核心问题：

1. 一个视频里有哪些人物。
2. 每个人在什么时候出现。
3. 多个视频/多部作品里哪些人物是同一个人。
4. 如何把可用参考帧整理出来，给后续 AI 生成角色设定图、四视图、人物档案使用。

## 目标用户

| 用户 | 典型场景 | 成功标准 |
| --- | --- | --- |
| 动漫/影视素材整理者 | 批量分析多集视频，整理人物出现时间 | 快速看到人物清单和时间轴 |
| 剪辑辅助用户 | 找某个人出现的片段 | 点人物/片段即可跳转播放器 |
| 人物档案整理者 | 为角色建立跨作品档案和参考素材 | 能按人物/按作品整理、合并、重命名 |
| AI 参考图生产者 | 从视频帧选素材，生成角色三视图/四视图 | 能导出完整画面、脸部标记和四视图资产 |
| 脚本用户 | 批量跑分析、读取结果 | CLI/API 可稳定调用 |

## 产品原则

1. **本地优先**：视频、数据库、截图和 API Key 都在项目本地，默认不依赖云端。
2. **分析与整理分离**：模型负责初筛，人负责最后确认；不要试图把误差藏起来。
3. **像剪辑软件一样工作**：左侧素材，中间播放器/时间轴，右侧检查器；常用操作直接可见，低频操作收纳。
4. **按作品一定正确，按人物逐步修正**：按作品查看不会错；跨作品人物自动聚类只是初稿，需要合并/拆分/重命名。
5. **参考素材服务后续 AI**：完整画面、脸部标记、脸部截图都必须保留，因为多模态模型需要上下文。
6. **配置集中可读**：识别、输出、参考图、AI 参数都写在 TOML，并带中文注释。
7. **不做重型标注平台**：不做训练集管理、不做复杂审核流；只做当前任务需要的修正能力。

## 信息架构

当前产品有三个主页面：

```text
工作台 /workspace
  左侧：素材、导入、任务
  中间：视频、人物时间轴、批量整理人脸
  右侧：作品信息、人物清单、快速操作、档案关联

人物档案 /profiles
  左侧：按人物 / 按作品 / 回收站、搜索、新建档案
  中间：人物或作品详情、参考帧、四视图资产
  右侧：人物片段预览、参考素材、档案管理

搜索 /search
  左侧：关键词、以图搜图、阈值
  中间：文本结果
  右侧：图片匹配结果
```

## 页面设计

### 工作台

![工作区](assert/工作区1.webp)

工作台是高频页面，目标是分析和修正单个视频。

关键设计：

- 顶部显示当前视频统计：视频数、人物数、片段数、当前播放时间。
- 左侧素材列表必须可滚动，能看到所有导入项和任务状态。
- 中间播放器优先级最高，红框只是辅助，不应遮挡主要操作。
- 人物时间轴默认显示全部视频范围，并支持缩放和平移。
- 右侧人物清单支持选中/取消高亮、隐藏、删除、档案关联。
- 批量整理人脸需要保留底部空间，避免最后一行图片被屏幕裁掉。

### 人物档案

![人物档案按作品](assert/人物档案1.webp)

人物档案页解决跨视频整理问题。这里的设计重点不是“显示更多卡片”，而是让用户能判断：

```text
这个 person 到底是不是同一个角色？
这组参考帧能不能用于生成角色设定图？
这个四视图资产属于哪个人物、哪个形象？
```

按作品模式：

- 一部作品下列出所有本地人物。
- 每个人横向展示基础信息、候选帧和四视图资产。
- 支持全选此人物的参考帧，方便新建档案或生成四视图。
- 右侧片段预览固定在顶部附近，长列表滚动时仍能看到视频。

按人物模式：

- 一个全局人物下按作品列出观测。
- 可以选择某个作品中的全部参考帧。
- 可以上传/生成多张四视图资产。
- 可以重命名、合并、删除档案。

回收站：

- 删除不是直接不可逆。
- 用户可以恢复误删档案。
- 彻底删除需要明确确认。

### 搜索

![人物搜索](assert/人物搜索.webp)

搜索页用于在已有数据库中找人。

设计要点：

- 文本搜索和以图搜图并列，但结果区域分开。
- 图片搜索必须显示上传的图片文件名和匹配列表。
- 以图搜图阈值可调，用户能自己收紧或放宽结果。
- 搜索结果应保留视频名、人物名、全局人物 ID、片段数和相似度。

## 人物档案模型

人物有三层身份：

| 层级 | 示例 | 说明 |
| --- | --- | --- |
| 本地人物 | `person_002` | 单个视频内聚类出的结果 |
| 全局人物 | `global_person_000012` | 跨视频聚合的人物档案 |
| 作品形象 | `形象 01` | 某个作品/服装/状态下的参考资产集合 |

设计约束：

1. 一个本地人物只能属于一个全局人物。
2. 一个全局人物可以有多部作品的观测。
3. 一个全局人物可以有多张四视图原图。
4. 四视图资产可以来自用户上传，也可以来自 AI 生成。
5. 当用户从错误聚类中拆出新档案时，相关四视图资产应允许迁移。

## 四视图资产设计

四视图不强制切成四张图。当前以“一张未切分原图”为资产单位，默认更偏向横向角色设定表，而不是 2x2 方格图。

![四视图资产示例](assert/四视图2.webp)

默认内容：

```text
正面全身 / 侧面全身 / 背面全身 / 45 度全身或半身 / 右侧细节区
```

右侧细节区建议做成两列：

- 第一列是脸部细节：45 度脸部特写、2~3 个轻微微表情小头像。
- 第二列是设计细节：服装、发型、配饰、材质、纹样等关键设计。

为什么不是只生成四个方向：

- 动漫角色更需要脸部、发型、服装细节。
- 45 度特写和微表情对后续多模态理解更有价值。
- 同一人物可能有多个造型，资产必须允许多张。

资产展示原则：

- 默认直接展示未切分原图，点击后放大查看。
- 一个人可以挂多张四视图资产，用于不同服装、形象或 AI 生成版本。
- 空状态也要保留上传和生成入口，人物档案不依赖视频分析结果。
- 输出格式默认 WebP，减少本地档案体积；需要最大兼容性时可切到 PNG。

## 参考素材设计

参考素材分三种视图：

| 视图 | 价值 |
| --- | --- |
| 完整画面 | 看身体、服装、场景、多人站位 |
| 脸部标记 | 在完整画面中明确“目标是谁” |
| 脸部截图 | 快速比对脸部身份 |

默认优先完整画面。脸部截图适合快速筛选，但给 AI 生成角色设定图时，完整画面和脸部标记通常更可靠。

## 识别质量设计

产品不承诺自动识别 100% 准确。当前设计是“先自动粗分，再人工细调”。

主要误差来源：

- 低分辨率小脸。
- 动漫脸和真人模型分布差异。
- 大幅动作、遮挡、侧脸。
- 背景纹理误检。
- 同一角色换装或表情夸张导致 embedding 不稳定。

产品上的解决方式：

1. 配置文件提供召回/严格两种预设。
2. 工作台提供删除误检、归类、合并、隐藏。
3. 人物档案提供合并、重命名、回收站。
4. 参考素材保留脸部标记，降低 AI 后续误读。

## 后续产品方向

1. 人物档案中增加“形象”实体，把四视图和参考帧更明确地绑定到服装/造型。
2. 搜索结果点击后跳转到工作台或人物档案对应位置。
3. 参考素材导出为可直接给多模态模型使用的压缩包和 manifest。
4. 识别质量评测面板：展示 raw faces、kept faces、tracks、people、误检率。
5. 更多手动纠错入口：手动框选一张脸并创建轨迹/人物。

## 文案和术语

| 术语 | 使用方式 |
| --- | --- |
| 作品 | 一个视频或一组同源视频的展示名 |
| 人物 | 单个视频内识别出的本地人物 |
| 人物档案 | 跨视频的全局人物 |
| 参考素材 | 用于人工判断或 AI 生成的帧图 |
| 四视图资产 | 一张未切分的角色设定图原图 |
| 回收站 | 可恢复删除，区别于彻底删除 |

UI 文案原则：

- 操作按钮用动词：生成、上传、删除、恢复、合并。
- 危险操作明确后果并二次确认。
- 不使用“识别 4:10”这类模糊说法，耗时统一写“耗时”。
