# 批量整理人脸交互问题记录

最后更新：2026-07-07

## 页面位置

工作台 `/workspace` 中间下方的“批量整理人脸”区域。

它的目标是修正自动识别结果：

- 多个 track 被分成了同一个人或不同人。
- 背景墙、局部眼睛、纹理被误识别人脸。
- 角色短暂出场被分错。
- 用户需要快速把一组截图归类到目标人物。

## 当前交互规则

| 操作 | 当前设计 |
| --- | --- |
| 单击卡片 | 选中/取消 |
| 空白处拖动 | 框选 |
| 多选后拖动抓手 | 拖动整组 |
| 拖到右侧人物清单 | 把选中轨迹归类到该人物 |
| 拖到时间轴人物轨道 | 把选中轨迹归类到该轨道人物 |
| 删除人物/轨迹 | 危险操作，必须确认 |
| 取消选择 | 清空已选 |

关键原则：

```text
卡片主体负责选择；抓手负责拖动；删除不绑定普通点击。
```

## 当前仍需观察

1. 触摸板滚动和拖动是否冲突。
2. 多选很多张时，拖动反馈是否足够清楚。
3. 长视频大量候选图时，列表性能是否下降。
4. 时间轴轨道作为 drop target 时，命中区域是否足够大。
5. 删除后时间轴和人物清单是否所有状态同步刷新。

## 交互约束

不要把这些操作绑定到普通点击：

- 删除。
- 合并。
- 归类。
- 彻底删除。

原因：这些区域同时承担选择、预览、拖动和播放定位，普通点击太容易误触。

危险操作推荐：

```text
按钮/菜单 -> 确认弹窗 -> 执行 -> 刷新人物/时间轴/参考素材
```

## 和人物档案的关系

批量整理人脸只修正“当前视频内”的本地人物和轨迹。

人物档案页修正的是“跨作品人物档案”：

- 新建全局档案。
- 合并全局档案。
- 重命名全局档案。
- 迁移四视图资产。
- 删除/恢复档案。

不要把两个层级混在一个操作里。工作台适合修本地识别结果；人物档案适合修跨作品归类。
