# 项目完整性评估

最后更新：2026-07-07

## 当前结论

FaceTimeMarker 已经从“单视频分析工具”演进到“本地人物档案工作台”阶段。

当前已具备完整闭环：

```text
导入视频
  -> 后台分析
  -> 写入数据库
  -> 工作台复核
  -> 人物档案整理
  -> 搜索人物
  -> 生成/上传四视图资产
```

仍然不是成熟商用级人物系统。下一阶段重点应是识别质量评测、档案形象模型、长视频稳定性和数据迁移，而不是继续无节制增加新页面。

## 当前强项

1. **本地闭环完整**：不依赖云端即可完成导入、分析、整理、搜索。
2. **产品方向清晰**：剪辑软件式工作台 + 人物档案系统。
3. **配置可读**：TOML 拆分后，识别、输出、AI、参考图配置更集中。
4. **人工整理入口足够**：删除、合并、隐藏、重命名、归类和回收站已覆盖主要误差。
5. **AI 参考资产闭环**：可从视频参考帧生成四视图，并能删除/迁移资产。

## 主要短板

| 问题 | 影响 | 建议 |
| --- | --- | --- |
| 动漫识别仍不稳定 | 主角漏检、背景误检、人物分裂 | 建立固定样例集和参数评测表 |
| 人物档案缺少“形象”实体 | 多套服装/造型只能靠多张资产堆叠 | 增加 appearance/profile asset group |
| 以图搜图准确率不稳定 | 返回不相关结果 | 增加阈值说明、质量评分、候选过滤 |
| 时间轴和代表帧仍受采样率影响 | 用户感觉片段或头像偏移 | UI 明确采样帧逻辑，支持更高采样配置 |
| 源媒体丢失缺少重链接 | 原路径移动后播放/截帧失败 | 做源媒体重新链接 |
| 长视频性能评估不足 | 无法估算不同机器耗时 | 固定 benchmark 视频和报告格式 |
| 前端部分页面仍偏大 | 后续维护成本高 | 按业务面板继续拆组件/组合函数 |

## 下一阶段建议

优先级从高到低：

1. 建立识别评测集：真人、动漫、低清、多人同屏各 3-5 个短视频。
2. 做人物“形象”模型：一个人物可有多个作品形象，每个形象有参考帧和四视图资产。
3. 做源媒体重链接：源文件丢失时像剪辑软件一样重新绑定路径。
4. 完善以图搜图质量：阈值、质量评分、结果解释、空结果状态。
5. 继续前端拆分：`WorkspacePage.vue` 可进一步拆组合逻辑和业务面板。
6. 做数据迁移规范：SQLAlchemy 模型、Alembic、旧库兼容测试。

## 不建议现在做

- 直接引入人体检测模型。
- 立刻换成远程数据库。
- 做云端账号和权限。
- 把人物档案做成复杂 CRM。
- 继续增加页面但不整理现有组件。
